THE 6 VERIFICATION CHECKPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CHECKPOINT 1: SOURCE WHITELIST
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Verify the cited source is in our verified corpus
TRIGGER:     Every output citation
ACTION:      BLOCK if source not in whitelist
EXAMPLE:
  ✗ Output claims: "According to the Islamic University of Madinah, Finance Paper Vol. 7 2021"
  × Not in whitelist
  → BLOCK: "Source not verified"
──────────────────────────────────────────────────────────────────────────────

CHECKPOINT 2: CITATION EXISTENCE
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Verify the specific citation exists in the database
TRIGGER:     Every reference (verse, hadith, resolution, standard)
ACTION:      BLOCK if citation doesn't exist
EXAMPLES:
  ✗ "Quran 2:300" → Does Surah 2 have 300 ayahs? NO (max 286) → BLOCK
  ✗ "Bukhari 5432" → Does this hadith number exist? NO → BLOCK
  ✗ "IIFA Resolution 108 on Cryptocurrency" → Session 0, Res 108? NO → BLOCK
  ✗ "Fiqh al-Zakat Vol. 3" → Only 2 volumes exist → BLOCK
  ✓ "Quran 2:275" → Surah 2, Ayah 275 exists → PASS
──────────────────────────────────────────────────────────────────────────────

CHECKPOINT 3: TEXT HASH MATCH
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Verify the quoted text matches the database exactly
TRIGGER:     Every direct quotation
ACTION:      BLOCK if text doesn't match SHA-256 hash
EXAMPLE:
  Original in DB: "المال والبنون زينة الحياة الدنيا"
  Hash stored:    sha256:xyz789abc...

  System tries to output: "المال والبنون هما زينة الحياة الدنيا" (added "هما")
  New hash:        sha256:different_hash...
  → BLOCK: "Text mismatch detected"
──────────────────────────────────────────────────────────────────────────────

CHECKPOINT 4: GRADING AUTHORITY
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Verify hadith grades come from recognized scholars only
TRIGGER:     Every hadith grading claim
ACTION:      FLAG if grader not in approved list

APPROVED GRADERS ONLY:
  ✓ Al-Albani          (1914-1999)  — Silsilat al-Ahadith al-Sahihah
  ✓ Imam Nawawi        (1233-1277)  — Classical authority
  ✓ Ibn Hajar          (1372-1449)  — Fath al-Bari
  ✓ Ahmad Shakir       (1892-1958)  — Quranic scholar
  ✓ Abd al-Rahman al-Arna'ut (modern) — Hadith expert

REJECTED GRADERS:
  ✗ "Brother Ahmed (YouTube)"      → Not qualified
  ✗ "Anonymous blogger"             → No accountability
  ✗ AI-generated grading            → Not transmitted scholarship
  ✗ Unverified modern scholars      → No ijaza

EXAMPLE:
  ✗ Claim: "Hadith graded Sahih by Brother Ahmed"
  → Not in approved list → FLAG: "Grading Status: Unverified"
──────────────────────────────────────────────────────────────────────────────

CHECKPOINT 5: UNCERTAINTY FLAG
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Flag outputs with low confidence
TRIGGER:     Confidence score < 95%
ACTION:      FLAG with disclaimer about scholarly disagreement

CONFIDENCE LEVELS:
  HIGH (95-100%):    Consensus (Ijma') or explicit Quranic verse
                     Display: "This is the position of all major scholars"

  MEDIUM (70-94%):   Majority view with minority positions
                     Display: "⚠️ The majority of scholars hold this view.
                               Some scholars differ."

  LOW (<70%):        Scholarly disagreement (Ikhtilaf)
                     Display: "⚠️ SCHOLARLY DISAGREEMENT - Multiple schools
                               hold different positions on this matter."

EXAMPLE:
  Question: "What do Islamic scholars say about [controversial topic]?"
  Confidence: 45% (because different schools disagree)
  → FLAG: Display all positions with clear "Ikhtilaf" label
──────────────────────────────────────────────────────────────────────────────

CHECKPOINT 6: FATWA BOUNDARY
──────────────────────────────────────────────────────────────────────────────
FUNCTION:    Prevent AI from issuing binding legal rulings
TRIGGER:     Personal obligation/ruling requests
ACTION:      REDIRECT to qualified scholar

FATWA INDICATORS:
  ✗ "Should I..."          → Personal obligation query
  ✗ "Is it halal/haram?"   → Binary ruling request
  ✗ "What is the ruling?" → Legal judgment
  ✗ "Am I allowed to...?"  → Personal permission request

RESPONSE:
  "This system provides research information, not binding religious rulings.

  For a personal fatwa, consult:
  • Dar al-Ifta' al-Misriyyah (Egypt)
  • Your local Imam or Islamic scholar
  • Regional fatwa council"

EXAMPLE:
  User: "Should I wear gold? Is it halal for men?"
  → Fatwa boundary detected
  → REDIRECT with contact info for qualified muftis
──────────────────────────────────────────────────────────────────────────────
